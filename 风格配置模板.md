## 预设风格配置

### 1. 老司机风格

```yaml
style_name: "老司机风格"
description: "接地气、犀利、反常识的经验分享风格"

language:
  formality: 2          # 非常口语化
  humor: 8              # 幽默感强
  emotion: 7            # 情感浓度高
  directness: 9         # 非常直接

structure:
  opening_style: "冲突"
  paragraph_length: "短"
  use_subheadings: true
  use_lists: true
  use_bold: true

content:
  theory_ratio: 20      # 理论占20%
  case_ratio: 50        # 案例占50%
  data_ratio: 10        # 数据占10%
  story_ratio: 20       # 故事占20%

rhetoric:
  use_metaphor: true
  use_repetition: true
  use_contrast: true
  use_questions: true

vocabulary:
  common_phrases:
    - "兄弟们"
    - "我跟你说"
    - "说实话"
    - "这特么"
    - "你猜怎么着"
  
  sentence_starters:
    - "很多人觉得……但我要告诉你"
    - "说实话"
    - "我见过太多"
    - "这就是为什么"
  
  transition_words:
    - "但问题来了"
    - "更关键的是"
    - "说到这里"
    - "举个例子"

tone:
  - 直接犀利
  - 不绕弯子
  - 有点愤世嫉俗
  - 但真诚可靠
```

## 自定义风格模板

```yaml
style_name: "我的风格"
description: "在这里描述你的风格特点"

# 语言特征（0-10分）
language:
  formality: 5          # 正式程度（0=极口语，10=极正式）
  humor: 5              # 幽默程度（0=严肃，10=搞笑）
  emotion: 5            # 情感浓度（0=冷静，10=激情）
  directness: 5         # 直接程度（0=委婉，10=直白）

# 结构特征
structure:
  opening_style: "故事"  # 开头方式：悬念/故事/观点/数据/问题/冲突
  paragraph_length: "中" # 段落长度：短/中/长
  use_subheadings: true  # 是否使用小标题
  use_lists: true        # 是否使用列表
  use_bold: true         # 是否使用加粗

# 内容特征（总和=100%）
content:
  theory_ratio: 30      # 理论占比（%）
  case_ratio: 40        # 案例占比（%）
  data_ratio: 20        # 数据占比（%）
  story_ratio: 10       # 故事占比（%）

# 修辞特征
rhetoric:
  use_metaphor: true    # 使用比喻
  use_repetition: true  # 使用重复
  use_contrast: true    # 使用对比
  use_questions: true   # 使用反问

# 词汇特征
vocabulary:
  common_phrases:       # 常用短语
    - "你的常用语1"
    - "你的常用语2"
    - "你的常用语3"
  
  sentence_starters:    # 句子开头
    - "你常用的开头1"
    - "你常用的开头2"
  
  transition_words:     # 过渡词
    - "你的过渡词1"
    - "你的过渡词2"

# 整体基调
tone:
  - "特点1"
  - "特点2"
  - "特点3"

# 参考对象（可选）
reference:
  authors:              # 参考作者
    - "作者1"
    - "作者2"
  
  articles:             # 参考文章
    - "文章链接1"
    - "文章链接2"

# 禁忌（可选）
avoid:
  words:                # 避免使用的词
    - "禁用词1"
    - "禁用词2"
  
  patterns:             # 避免的模式
    - "不要过度使用感叹号"
    - "不要使用网络烂梗"
```

## 风格调试指南

### 如何测试你的风格

1. **写一段200字的测试文本**
   - 用你的风格配置生成
   - 读一遍，看是否符合预期

2. **调整参数**
   - 太正式？降低 formality
   - 太严肃？提高 humor
   - 太冷淡？提高 emotion
   - 太啰嗦？提高 directness

3. **对比测试**
   - 同一主题，用不同参数生成
   - 看哪个版本更符合你的预期

4. **读者测试**
   - 给3-5个朋友看
   - 收集反馈
   - 调整参数

### 常见问题

**Q：我的风格太平淡怎么办？**
A：提高以下参数：
- humor +2
- emotion +2
- use_metaphor = true
- use_contrast = true

**Q：我的风格太浮夸怎么办？**
A：降低以下参数：
- humor -2
- emotion -2
- directness -1
- 减少感叹号和夸张词

**Q：我的风格太啰嗦怎么办？**
A：调整以下参数：
- directness +2
- paragraph_length = "短"
- 提高 data_ratio，降低 story_ratio

**Q：我的风格太干巴巴怎么办？**
A：调整以下参数：
- emotion +2
- story_ratio +10
- use_metaphor = true
- 增加个人经历

## 风格进化建议

### 阶段1：模仿期（0-3个月）
- 选择1-2个预设风格
- 严格按照配置写作
- 不要急于创新

### 阶段2：调整期（3-6个月）
- 根据反馈微调参数
- 尝试混合风格
- 找到舒适区

### 阶段3：形成期（6-12个月）
- 形成自己的风格配置
- 可以灵活调整
- 但保持核心特征

### 阶段4：成熟期（12个月+）
- 风格稳定但不僵化
- 根据主题灵活调整
- 形成个人品牌

---

## 最后的建议

1. **不要追求完美的风格**
   - 风格是演化的，不是设计的
   - 重要的是真诚，不是完美

2. **不要频繁更换风格**
   - 读者需要时间适应
   - 频繁更换会失去辨识度

3. **不要完全依赖配置**
   - 配置是辅助，不是替代
   - 你的思考才是核心

4. **不要害怕调整**
   - 根据反馈持续优化
   - 没有一成不变的风格

**记住：最好的风格，是最适合你的风格。**
